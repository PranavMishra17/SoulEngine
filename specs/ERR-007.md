# ERR-007: CSS design-token bugs and permanent guard

## Problem
Two categories of silent CSS token bugs allow brand-color drift and broken styles:

1. `--accent-primary-rgb` is referenced ~12 times across `pages.css` and `pages-app.css` with fallback `99,102,241` (old indigo) but is never defined, so landing page accents render indigo instead of ember brand color (`#e07850`).
2. `conversation-modes.css` references 6 phantom tokens (`--bg-secondary`, `--border-primary`, `--bg-primary`, `--text-primary`, `--accent-primary`) that don't exist — the real names are `--color-bg-secondary`, `--color-border`, etc.

## Acceptance Criteria
1. Define `--accent-primary-rgb: 224, 120, 80;` (ember RGB matching `--color-accent-primary`) in `:root` in `web/css/design-system.css` so all rgba() references compute to brand color
2. Replace all phantom token references in `conversation-modes.css` with the real `--color-*` tokens
3. Add permanent regression test `tests/unit/err-007-css-tokens.test.ts` that scans all CSS files, extracts defined tokens and unfallback-guarded references, and asserts no undefined tokens are used
4. `npm run build` (typecheck) and `npm test` both green

## Approach
- Add `--accent-primary-rgb` to the `:root` block in `design-system.css` (line ~10-148)
- Update 6 token references in `conversation-modes.css` (lines 6, 7, 18, 20, 158, 162)
- Write a Vitest test that reads all `web/css/**/*.css` files, strips CSS comments, extracts `--*:` definitions vs `var(--*)` references without fallbacks, and fails if any undefined tokens are referenced

## Files to Touch
- `web/css/design-system.css` — add `--accent-primary-rgb` definition
- `web/css/conversation-modes.css` — replace 6 phantom token references
- `tests/unit/err-007-css-tokens.test.ts` — new regression test

## Test Plan
1. Before fix: test fails, identifying both `--accent-primary-rgb` (referenced but undefined) and the 6 phantom tokens
2. After fix: test passes (all referenced tokens are defined)
3. Visual: landing page accents render ember orange, not indigo
4. Full suite green
